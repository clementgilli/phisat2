Training pipeline

End-to-end flow used by Make targets and CLI.

DocsTraining

Execution path

  1. Make target calls python -m phisat2.cli.train fit ....
  2. CLI parses command and validates task / dataset / model / dataloader.
  3. resolve_task_spec sets task metadata including output size and target key.
  4. For each seed: set RNG seeds, build datamodule, build model, wrap with LightningModule.
  5. Lightning Trainer runs fit with callbacks + logger + hardware config.

Training step

  • Image is passed to the model from batch key image.
  • Target is batch[spec.target_key] (mask/label/target).
  • Loss: cross-entropy for segmentation/classification, MSE for regression-like outputs.
  • Spatial losses interpolate prediction to target resolution if needed.

Defaults and hardware

  • Model + optimizer: frozen model.encoder, trainable non-encoder layers, AdamW + weight decay 1e-4.
  • Optimizer LR source: Make LR or CLI --lr.
  • Checkpointing saves best (monitor val_loss) and last.
  • Fast-dev mode disables checkpointing and uses one-cycle validation behavior.

Auto-DDP helper converts to DDP only when --auto-ddp and multiple CUDA devices are available.

Common Make entry points

make fast-dev-run DATASET=clouds MODEL=phisat2_geoaware DATALOADER=synthetic SEEDS=0
make train TASK=classification MODEL=terramind_v1_tiny DATASET=climate DATALOADER=h5_pairs SEEDS="13 42" EPOCHS=10
make smoke
